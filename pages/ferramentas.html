INTRODUÇÃO À PROGRAMAÇÃO COMPETITIVA

Ambiente BOCA

O BOCA (BOCA Online Contest Administrator) é uma solução completa para gerenciar competições de programação, como a renomada Maratona de Programação da Sociedade Brasileira de Computação (SBC). Desenvolvido em PHP e projetado para operar em ambiente web, o BOCA permite interações ágeis via navegador, otimizando a submissão e correção de problemas tanto em competições quanto em contextos acadêmicos.

Com uma interface simples, funcional e personalizável, o BOCA se adapta às necessidades de diversas instituições e equipes organizadoras. Ele é compatível com Linux e pode ser facilmente executado em containers Docker, oferecendo flexibilidade e praticidade para diferentes ambientes de uso.

Quer explorar o potencial dessa ferramenta? Confira nosso tutorial e descubra como instalar, configurar e aproveitar ao máximo o BOCA para elevar a qualidade de suas competições e atividades acadêmicas!

Como Instalar o BOCA

Confira os passos necessários para instalar o BOCA em diferentes sistemas operacionais.

1. Pré-requisitos

Antes de instalar o BOCA, certifique-se de atender aos seguintes pré-requisitos, os quais ensinamos como obtê-los:

  • Caso utilize Windows, tenha o WSL 2 com o Ubuntu;
  • Tenha o Docker Engine instalado no Ubuntu (no WSL 2 ou no nativo).

Se você já atende aos dois, então você pode pular para a instalação do BOCA, mas caso não atenda algum deles continue a leitura do tutorial.

Existem duas maneiras para obter o Docker no Windows: o Docker Desktop e o Docker Engine. Ambas as opções requerem o uso do WSL 2 (Subsistema do Windows para Linux), que permite executar uma distribuição Linux sem a necessidade de uma máquina virtual. Este tutorial usará o Docker Engine, mais próximo do Linux nativo.

A seguir, os requisitos para instalar o WSL 2.

Requisitos mínimos para o WSL 2:

  • Windows 10 versão 2004 ou superior;
  • Windows 11 versão 22000 ou superior;
  • 4 GB de memória RAM (recomendado: 8 GB);
  • Processador compatível com virtualização.

Observação: Para obter uma performance estável, mantenha o Windows atualizado, pois as dependências do WSL 2 são frequentemente aprimoradas pelo Windows Update.

Instalação do WSL 2 (apenas para usuários de Windows)

No PowerShell, siga os passos:

No Windows 10 a partir da versão 2004 ou no Windows 11, o WSL já está instalado. Para atualizar o WSL, execute:


                        > wsl --update

Atribua a versão 2 como padrão do WSL. Execute:


                        > wsl --set-default-version 2

Iremos instalar o Ubuntu, distribuição padrão do WSL. Para isso, execute o seguinte comando:


                        > wsl --install

Após instalar o Ubuntu, abrirá um novo terminal, sendo o próprio ambiente Linux, pedindo o nome de usuário (Unix username) e a senha (password). Cuidado para não colocar uma senha que vá esquecer, pois ela será necessária em outros momentos.

Caso não apareça o novo terminal, provavelmente sua máquina estará com a opção de virtualização desativada. Nesse caso, procure em um tutorial específico para sua placa-mãe (ou no próprio manual que vem com ela) sobre como ativar a virtualização.

Para acessar o Linux, na barra “Pesquisar” do Windows, digite “Ubuntu” e tecle Enter.

Agora que os usuários de Windows possuem o Ubuntu a partir do WSL 2, a instalação do Docker será a mesma para quem utiliza Windows e para quem usa Linux. Confira a seguir:

2. Instalando Docker (válido para Windows e Linux Ubuntu)

Caso esteja no Ubuntu, entre no terminal. Caso esteja usando o Windows, na barra de pesquisa procure por “Ubuntu” e tecle Enter. Independentemente do seu sistema operacional, siga os passos abaixo.

Para evitar conflitos de pacotes, execute o comando abaixo. Será preciso inserir a sua senha de usuário do Ubuntu, que foi configurada durante a instalação:


                        $ for pkg in docker.io docker-doc docker-compose docker-compose-v2 podman-docker containerd runc; do sudo apt-get remove $pkg; done

Antes de instalar o Docker, execute o seguinte comando para configurar sua máquinaa:


                        $ 
 
                            sudo apt-get install ca-certificates curl 

                            sudo install -m 0755 -d /etc/apt/keyrings 

                            sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc 

                            sudo chmod a+r /etc/apt/keyrings/docker.asc 

                            echo \ 

                            "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \ 

                            $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \ 

                            sudo tee /etc/apt/sources.list.d/docker.list > /dev/null 

                            sudo apt-get update

Para instalar a versão mais recente do Docker, execute (observação: se for perguntado algo como: “After this operation, some additional disk space will be used. Do you want to continue? [Y/n]”, tecle Y e Enter):

sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

Verifique se a instalação foi bem-sucedida executando a imagem hello-world:

sudo docker run hello-world

Deverá aparecer uma mensagem semelhante a essa:



Pronto, você instalou o Docker com sucesso!


Caso não tenha sido bem-sucedido até aqui, confira a seção de Perguntas Frequentes.

Se você seguiu os passos anteriores, agora os dois arquivos estão em uma pasta (aqui denominada “boca”) no seu diretório “C:\”. Simplificadamente, o procedimento é acessar o diretório “C:\” do Windows, copiar a pasta “boca” e colar no diretório “home” do Ubuntu.



No terminal Ubuntu, acesse o diretório do Windows onde os arquivos foram salvos:

cd /mnt/c

Naturalmente, “cd” é o comando do bash do Linux para mudar de diretório. Com o “/mnt/c”, cria-se um ponto de montagem do sistema de arquivos do Windows, possibilitando o acesso do Ubuntu aos arquivos.

Copie a pasta “boca” para o diretório “home” do Ubuntu com o seguinte comando: (substitua “usuario” pelo nome do seu usuário Linux)

sudo cp -r /mnt/c/boca /home/usuario

Agora você já tem acesso à pasta “boca” pelo próprio Ubuntu. Acesse-a com:

cd /home/usuario/boca

Agora que estamos dentro do diretório da pasta “boca” no Ubuntu, execute o comando:

sudo docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

Criação e Ativação de um Contest no BOCA

Este tutorial apresenta um guia detalhado para criar e ativar um contest no sistema BOCA (BOCA Online Contest Administrator). Siga os passos abaixo para configurar seu contest com sucesso.

1. Acessando a Aba Contest

  1. Faça login utilizando as credenciais system e boca.
  2. Clique em "Options" e, por segurança, altere a senha do usuário "system" para uma senha segura.
  3. No menu principal, clique na aba Contest e selecione a opção New para iniciar a criação de um novo contest.

2. Configurando o Novo Contest

Altere ou configure as informações necessárias para o novo contest nos campos fornecidos. Abaixo está uma descrição detalhada dos principais campos:

  • Name: Defina o nome do contest (por exemplo, "Maratona de Programação 2025").
  • Start date: Escolha a data e o horário em que o contest começará.
  • Duration: Insira a duração do contest em minutos (exemplo: 300 minutos para 5 horas).
  • Stop answering: Especifique o número de minutos a partir dos quais as equipes não receberão feedback sobre suas submissões. Valor usual: 285 minutos, ou seja, ficam os 15 minutos finais sem respostas.
  • Stop scoreboard: Insira o número de minutos a partir dos quais o placar será congelado para manter o vencedor em segredo. Valor usual: 240 minutos.
  • Penalty: Configure a penalidade em minutos por submissões incorretas. Valor usual: 20 minutos.
  • Max file size allowed for teams: Determine o limite de tamanho para os arquivos de código-fonte enviados. Esta definição é por motivos de segurança.
  • Contest main site number:: Para contests multisite, insira o número atribuído ao site principal. Valor usual: 1.
  • Contest local site number:: Para contests multisite, insira o número atribuído ao site atual, conforme instruções do coordenador do contest.

3. Ativando o Contest

  1. Revise as informações configuradas.
  2. Clique no botão Send para salvar o novo contest.
  3. Em seguida, clique em Activate. Confirme a ação clicando em OK.
    • Atenção: Você será desconectado automaticamente ao ativar o contest.

4. Gerenciando o Contest

  1. Faça login novamente, agora utilizando as credenciais admin e boca.
  2. Clique em "Options" e, por segurança, altere a senha do usuário "admin" para uma senha segura.
  3. Aqui estão descrições sobre algumas opções do menu do BOCA para configurações adicionais dos contests:
    • Users: Adicione equipes e administradores.
    • Problems: Cadastre os problemas que serão resolvidos durante o contest.
    • Languages: Configure as linguagens de programação permitidas.
  4. Para mais orientações sobre como configurar seu contest, confira a documentação oficial do BOCA.

Considerações Finais

Este tutorial abordou o processo de criação e ativação de um contest no BOCA, incluindo a orientação sobre configurações adicionais. Certifique-se de seguir todas as instruções e revisar as configurações antes de ativar o contest para garantir que ele esteja alinhado às necessidades do evento.

Perguntas Frequentes

  • Como faço quando o container hello-world não é executado?

    1. Remova o Docker completamente. Para isso, execute: 
      sudo systemctl stop docker
sudo apt-get purge -y docker-engine docker docker.io docker-ce docker-ce-cli
sudo apt-get purge -y docker-ce docker-ce-cli
sudo apt-get autoremove -y
sudo apt-get autoclean
sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
sudo groupdel docker
sudo rm /usr/local/bin/docker
sudo rm /usr/local/bin/docker-compose
sudo rm /usr/local/bin/docker-machine
    2. Verifique se o Docker foi removido, executando o comando: 
      docker --version
    3. Instale o Docker novamente. Para isso, execute: 
      sudo apt-get update
sudo apt-get install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
    4. Teste a instalação criando novamente o container hello-world:
      sudo docker run hello-world
printf(“Seu foco determina sua realidade.”);
